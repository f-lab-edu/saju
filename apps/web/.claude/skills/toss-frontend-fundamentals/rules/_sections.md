# 섹션 정의

규칙 파일은 접두사로 섹션이 자동 분류됩니다.

## 섹션 목록

| 접두사 | 섹션 | 영향도 | 설명 |
|--------|------|--------|------|
| `readability-context-` | 가독성 > 맥락 줄이기 | CRITICAL | 불필요한 맥락을 제거하여 코드 이해 시간 단축 |
| `readability-naming-` | 가독성 > 이름 붙이기 | CRITICAL | 의미 있는 이름으로 코드 의도 명확화 |
| `readability-flow-` | 가독성 > 위에서 아래로 | CRITICAL | 자연스러운 읽기 흐름 유지 |
| `predictability-` | 예측 가능성 | HIGH | 함수/컴포넌트의 동작을 예측 가능하게 |
| `cohesion-` | 응집도 | HIGH | 함께 변경되는 코드를 함께 배치 |
| `coupling-` | 결합도 | MEDIUM | 코드 간 의존성을 최소화 |

## 규칙 파일 형식

각 규칙 파일은 다음 구조를 따릅니다:

```markdown
# 규칙 제목

## 요약
한 줄 요약

## 문제 상황
이 규칙이 필요한 이유

## 나쁜 예시
```code
// 개선 전 코드
```

## 좋은 예시
```code
// 개선 후 코드
```

## 핵심 포인트
- 포인트 1
- 포인트 2

## 참고
- 링크
```
