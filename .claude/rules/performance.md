# 성능 — 모델 선택 · 컨텍스트 관리

모델 선택 및 컨텍스트 윈도우 관리 규칙입니다.

## 모델 선택 전략

| 모델                   | 특징                         | 사용 시점                               |
| ---------------------- | ---------------------------- | --------------------------------------- |
| **Haiku 4.5**          | 가장 빠르고 저렴             | 자주 호출되는 경량 에이전트, 워커       |
| **Sonnet 5**           | 코딩 주력 모델               | 메인 개발, 멀티 에이전트 오케스트레이션 |
| **Opus 4.8**           | 깊은 추론, `/fast` 모드 지원 | 아키텍처 결정, 리서치·분석              |
| **Fable 5 (Claude 5)** | 최상위 지능 (Mythos-class)   | 가장 어려운 설계·검증 작업              |

- 서브에이전트는 기본 `model: inherit`로 두고, 역할이 명확히 가벼운 워커만 하위 모델을 지정한다.

## 컨텍스트 윈도우 관리

> ⚠️ 컨텍스트가 거의 찼을 때 복잡한 작업을 새로 시작하지 않는다.

- 사용량 확인은 `/context`.
- 컨텍스트가 길어지면 자동 요약(compaction)으로 작업이 이어지긴 하지만, 요약 전에 끝낼 수 있는 크기로 작업을 쪼개는 편이 안전하다.

### 남은 컨텍스트가 적을 때 피할 작업

- 대규모 리팩토링
- 여러 파일에 걸친 기능 구현
- 복잡한 상호작용 디버깅

### 컨텍스트 민감도가 낮은 작업

- 단일 파일 편집
- 독립적인 유틸리티 생성
- 문서 업데이트
- 간단한 버그 수정

## MCP 서버 관리

- MCP 도구 정의는 기본적으로 **지연 로드**된다 — 컨텍스트에는 도구 이름만 올라가고, 전체 스키마는 실제 사용 시점에 로드된다. 그래도 안 쓰는 서버는 꺼서 노이즈를 줄인다.
- `.mcp.json`의 특정 서버를 끄려면 settings.json에 `disabledMcpjsonServers`:

```json
{
  "disabledMcpjsonServers": ["server-1", "server-2"]
}
```

## Ultrathink + Plan Mode

깊은 추론이 필요한 복잡한 작업:

1. 프롬프트에 `ultrathink`를 포함하면 그 턴의 추론이 깊어진다 (`think hard` 등 다른 문구는 키워드로 인식되지 않음)
2. 구조화된 접근이 필요하면 **Plan Mode** 활성화
3. 독립적인 관점이 필요하면 역할 분리 서브에이전트 사용

## 빌드 문제 해결

빌드 실패 시:

1. `debugger` 에이전트로 원인 진단 (진단 전용 — 수정은 메인 대화에서)
2. 오류 메시지 분석
3. 점진적으로 수정하고, 각 수정 후 검증
