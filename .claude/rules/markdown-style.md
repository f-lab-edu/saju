# 마크다운 작성 스타일

`.md` 파일을 새로 작성하거나 수정할 때 지키는 스타일 규칙입니다.

## em-dash 금지

- em-dash(`—`, U+2014)를 쓰지 않는다. 대신 상황에 맞게:
  - 콜론(`:`)으로 부연 도입
  - 괄호(`( )`)로 보충 설명
  - 쉼표 또는 문장 분리로 흐름 정리
- en-dash(`–`)도 문장 구분 용도로는 쓰지 않는다. 숫자 범위는 물결표(`~`) 또는 `-`.

## 강조 표시는 최소한으로

- `**굵게**` / `*기울임*`은 정말 강조가 필요한 핵심 한두 곳에만 쓴다.
- 남용하면 강조가 강조가 아니게 된다. 목록 항목마다, 문장마다 굵게 처리하지 않는다.
- 순서 있는 목록(숫자 `1. 2. 3.` 또는 문자 `a. b.` / `가. 나.`로 정렬된 목록)의 항목에는 강조를 넣지 않는다. 번호 자체가 순서와 구조를 나타내므로 굵기가 불필요하다.
- 구조가 필요하면 강조 대신 제목(`##`), 목록, 표를 쓴다.

## 이모지 금지

- 파일을 작성하거나 수정할 때 이모지를 쓰지 않는다. 문서 본문, 제목, 목록, 표, 코드 주석, 커밋 메시지 어디에도 넣지 않는다.
- 경고나 강조가 필요하면 경고·체크·엑스 같은 기호 이모지 대신 텍스트로 표현한다 (예: "주의:", "필수", "금지").

## 적용 범위

- em-dash/강조 규칙은 Claude가 산출하는 마크다운 문서(문서 파일, README, 스킬 문서 등)에 적용된다.
- 이모지 금지는 마크다운뿐 아니라 Claude가 작성하는 모든 파일(코드, 설정, 커밋 메시지 포함)에 적용된다.
- 기존 문서를 대량으로 재작성하지는 않는다. 새로 쓰거나 해당 부분을 손댈 때 이 스타일을 따른다.
